# Rate Limiting & Auto 24-Hour IP Ban — `domain.com`

If a client sends more than **5 requests per second**, Nginx returns `429`. Fail2ban detects it and **blocks the IP via Nginx deny rules for 24 hours**.

---

## Step 1: Add One Line to `nginx.conf`

> `limit_req_zone` can only be defined in the `http` block (Nginx requirement). It does nothing until a domain explicitly references it. Other domains are unaffected.

Add this inside the `http { ... }` block in `/etc/nginx/nginx.conf`:

```nginx
limit_req_zone $binary_remote_addr zone=domain_rate_limit:10m rate=5r/s;
```

---

## Step 2: Create Fail2ban Blocked IPs File

```bash
sudo touch /etc/nginx/fail2ban_blocked.conf
```

> Fail2ban will automatically write `deny <ip>;` lines into this file when banning, and remove them when unbanning. **Do not edit this file manually.**

Full server block — save as `/etc/nginx/sites-enabled/domain.com.conf`:

```nginx

    set_real_ip_from 173.245.48.0/20;
    set_real_ip_from 103.21.244.0/22;
    set_real_ip_from 103.22.200.0/22;
    set_real_ip_from 103.31.4.0/22;
    set_real_ip_from 141.101.64.0/18;
    set_real_ip_from 108.162.192.0/18;
    set_real_ip_from 190.93.240.0/20;
    set_real_ip_from 188.114.96.0/20;
    set_real_ip_from 197.234.240.0/22;
    set_real_ip_from 198.41.128.0/17;
    set_real_ip_from 162.158.0.0/15;
    set_real_ip_from 104.16.0.0/13;
    set_real_ip_from 104.24.0.0/14;
    set_real_ip_from 172.64.0.0/13;
    set_real_ip_from 131.0.72.0/22;

        return 403;
    }

    # ── Rate Limiting — 5 req/sec, burst of 5, reject excess immediately ──
    limit_req zone=domain_rate_limit burst=5 nodelay;
    limit_req_status 429;



    # ROOT
    location = / {

    }

    location /redirect {

        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_pass_request_headers on;

        proxy_set_header X-JA3-Fingerprint $http_ssl_ja3;
        proxy_set_header X-JA3-Hash $http_ssl_ja3_hash;
        proxy_set_header X-JA4-Fingerprint $http_ssl_ja4;
        proxy_set_header X-H2-Fingerprint $http2_fingerprint;

        proxy_connect_timeout 60;
        proxy_send_timeout 60;
        proxy_read_timeout 60;
    }

    location /health {
        proxy_pass http://localhost:8003/health;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_pass_request_headers on;

        proxy_set_header X-JA3-Fingerprint $http_ssl_ja3;
        proxy_set_header X-JA3-Hash $http_ssl_ja3_hash;
        proxy_set_header X-JA4-Fingerprint $http_ssl_ja4;
        proxy_set_header X-H2-Fingerprint $http2_fingerprint;

        proxy_connect_timeout 60;
        proxy_send_timeout 60;
        proxy_read_timeout 60;
    }

    location ~* ^.*/(\.env.*|\.git.*|\.gitignore)$ {
        deny all;
        return 403;
    }

    access_log /var/log/nginx/domain_access.log fingerprint;
    error_log /var/log/nginx/domain_error.log;

    listen 443 ssl; # managed by Certbot
    ssl_certificate /etc/letsencrypt/live/domain.com/fullchain.pem; # managed by Certbot
    ssl_certificate_key /etc/letsencrypt/live/domain.com/privkey.pem; # managed by Certbot
    include /etc/letsencrypt/options-ssl-nginx.conf; # managed by Certbot
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem; # managed by Certbot
}

server {
    if ($host = domain.com) {
        return 301 https://$host$request_uri;
    } # managed by Certbot

    server_name domain.com;
    listen 80;
    return 404; # managed by Certbot
}
```

## Step 4: Fail2ban Setup

### Install
```bash
sudo apt update && sudo apt install fail2ban -y
```

### Create Nginx Block Action — `/etc/fail2ban/action.d/nginx-block.conf`

This action writes rules into a custom file used by Nginx's `geo` module (works behind Cloudflare/Azure where iptables cannot see the real client IP):

```bash
sudo tee /etc/fail2ban/action.d/nginx-block.conf > /dev/null << 'EOF'
[Definition]
actionban = echo "<ip> 1;" >> /etc/nginx/fail2ban_blocked.conf && nginx -t && nginx -s reload
actionunban = sed -i '/<ip> 1;/d' /etc/nginx/fail2ban_blocked.conf && nginx -t && nginx -s reload
EOF
```

### Create Filter — `/etc/fail2ban/filter.d/nginx-429.conf`
```bash
sudo tee /etc/fail2ban/filter.d/nginx-429.conf > /dev/null << 'EOF'
[Definition]
failregex = ^<HOST> -.* 429 
ignoreregex =
EOF
```

### Create Jail — `/etc/fail2ban/jail.d/nginx-domain.local`
```bash
sudo tee /etc/fail2ban/jail.d/nginx-domain.local > /dev/null << 'EOF'
[nginx-limit-req]
enabled  = true
port     = http,https
filter   = nginx-limit-req
logpath  = /var/log/nginx/domain_error.log
backend  = auto
action   = nginx-block
maxretry = 1
findtime = 5
bantime  = 86400

[nginx-429]
enabled  = true
port     = http,https
filter   = nginx-429
logpath  = /var/log/nginx/domain_access.log
backend  = auto
action   = nginx-block
maxretry = 1
findtime = 5
bantime  = 86400
EOF
```

## Step 5: Apply

```bash
# Test and reload Nginx
sudo nginx -t && sudo systemctl reload nginx

# Restart and enable Fail2ban
sudo systemctl restart fail2ban && sudo systemctl enable fail2ban

# Verify jails are active and using nginx-block action
sudo fail2ban-client status
sudo fail2ban-client get nginx-limit-req actions
sudo fail2ban-client get nginx-429 actions
```

## Step 6: Testing

### From an external machine (NOT the server):
```bash
# Send 20 parallel requests to trigger rate limit
for i in {1..20}; do
  curl -s -o /dev/null -w "Request $i: %{http_code}\n" https://domain.com/health &
done
wait

# Wait 5 seconds for Fail2ban to process, then try again
sleep 5
curl -s -o /dev/null -w "Blocked test: %{http_code}\n" --max-time 5 https://domain.com/health
# Expected: 403 (Nginx is denying the IP)
```

### On the server — verify the ban:
```bash
# Check if Fail2ban wrote the deny rule
cat /etc/nginx/fail2ban_blocked.conf

# Check ban status
sudo fail2ban-client status nginx-limit-req
sudo fail2ban-client status nginx-429
```

## Useful Commands

### Check Currently Banned IPs
```bash
sudo fail2ban-client status nginx-limit-req
sudo fail2ban-client status nginx-429
```

### Check Which IPs Are in the Nginx Block File
```bash
cat /etc/nginx/fail2ban_blocked.conf
```

### Unban a Specific IP
```bash
sudo fail2ban-client set nginx-limit-req unbanip <IP>
sudo fail2ban-client set nginx-429 unbanip <IP>
```

### Check Fail2ban Logs (Ban/Unban History)
```bash
# Full log
sudo tail -50 /var/log/fail2ban.log

# Only ban/unban events
sudo grep -E "Ban|Unban" /var/log/fail2ban.log | tail -20
```

```bash
# See rate-limited requests (error log)
sudo grep "limiting" /var/log/nginx/domain_error.log | tail -10

# See 429 responses (access log)
sudo grep " 429 " /var/log/nginx/domain_access.log | tail -10
```

```bash
sudo systemctl restart fail2ban
```

### Check Total Requests & Activity Time of Blocked IPs
You can run this script on the server to print a clean report of how many requests each blocked IP sent, and exactly when they started and stopped sending requests before getting blocked:

```bash
sudo bash -c '
echo -e "Blocked IP\tTotal Req\tFirst Request Time\t\tLast Request Time"
echo "--------------------------------------------------------------------------------------"
while read -r line; do
    # Extract the IP address from the line
    ip=$(echo "$line" | grep -oE "[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+")
    if [ ! -z "$ip" ]; then
        # Search Nginx logs for this IP (update domain_access.log to your specific log name if needed)
        log_entries=$(grep -w "$ip" /var/log/nginx/domain_access.log | awk "{print \$4}")
        if [ ! -z "$log_entries" ]; then
            total_req=$(echo "$log_entries" | wc -l)
            first_time=$(echo "$log_entries" | head -n1 | tr -d "[")
            last_time=$(echo "$log_entries" | tail -n1 | tr -d "[")
            echo -e "$ip\t$total_req\t\t$first_time\t$last_time"
        fi
    fi
done < /etc/nginx/fail2ban_blocked.conf
'
```
